# Agent Guide: Custom Expo Dev Client (Native Modules)

When building an Expo app that relies on **Native Modules** (like `expo-camera`, `@realm/react`, `expo-media-library`, or `expo-crypto`), standard **Expo Go** will often crash or throw "Native module not found" errors.

To properly simulate and develop these features, you must instruct the agent to use a **Custom Expo Dev Client**.

## 1. Setup the Environment

Instead of relying solely on standard Expo Go, run the following commands to install the required client library and generate the native Android/iOS project directories:

```bash
npx expo install expo-dev-client
npx expo prebuild --clean
```

## 2. Compile the Native App

Once the `android` and `ios` folders are generated by `prebuild`, you must compile the actual native application so the emulator/device has the binary containing the native modules.

For Android (requires Android Studio/SDK on the host machine):
```bash
npx expo run:android
```

For iOS (requires a Mac with Xcode):
```bash
npx expo run:ios
```

## 3. Run the Development Server

After the app is installed on the device/emulator, you no longer run `npx expo start`. Instead, you must run the server specifically telling it to connect to the custom dev client you just compiled:

```bash
npx expo start --dev-client
```

## 4. Re-compiling on Dependency Changes

**CRITICAL RULE:** If you install a *new* package that requires native code (e.g., adding `expo-location` or `expo-secure-store` later on), hot-reloading the JS will NOT work. You must re-run the compilation step:

```bash
npx expo prebuild --clean
npx expo run:android
```

## Summary for the Agent Prompt
When initiating a new project with an AI Agent, provide them with this context:

> *"Agent, this project uses native modules that are incompatible with standard Expo Go. You must use the Custom Expo Dev Client workflow. Install `expo-dev-client`, run `npx expo prebuild`, compile the app using `npx expo run:android`, and run the bundler with `npx expo start --dev-client`."*
